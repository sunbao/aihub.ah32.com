## Why

目前“创建用户”是匿名生成用户与 API 密钥：没有身份标识、也不利于后续运营与问题定位；同时用户在 UI 里看到一堆技术字段（例如各种 ID/UUID）会造成强烈噪音。

我们需要把“创建用户”打通 OAuth（首期 GitHub）：让用户用第三方身份完成创建，并把 GitHub 昵称/头像作为用户标识展示在控制台中。与此同时：
- 彻底取消匿名创建用户
- 任何编号/UUID 只属于后端日志与调试信息，UI 不展示、不回显、不要求用户记忆
- 控制台体验以手机端为主（触屏操作、可读性、少输入）

## Goals / Non-Goals

**Goals**
- 创建用户走 OAuth（首期 GitHub；后续可扩展其他提供方）
- 彻底取消匿名创建用户
- 不做组织/仓库成员限制，任何 GitHub 账号可用
- UI 显示 GitHub 昵称与头像，作为用户标识（中文界面，不混用中英文）
- 管理员权限仍然沿用 `AIHUB_ADMIN_TOKEN`（不与 GitHub 绑定）
- UI 不展示任何编号/UUID；成功/失败提示也不包含编号
- 前端风格以 `shadcn-ui/ui` 的组件与视觉规范为准，并优先保证手机端体验

**Non-Goals（本变更先不做）**
- 其他 OAuth 提供方的具体接入（本期只做 GitHub；但设计要保留扩展点）
- 组织/团队白名单限制
- 管理员体系重构（RBAC、多管理员）
- 全站改为 Cookie Session（本期保持“API Key 作为鉴权”模型）

## What Changes

- 新增 OAuth（GitHub）相关配置（client id/secret、回调地址等）
- 新增 OAuth 回调与用户绑定/创建逻辑（按第三方身份唯一）
- 删除匿名创建用户入口（UI 与 API）
- 更新 `/ui/user.html`：提供“用 GitHub 创建用户”，展示 GitHub 用户信息；不展示编号/UUID；手机端优先
- 更新文档与冒烟步骤（包含 OAuth 配置与回调验证）
