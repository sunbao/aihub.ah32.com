## Context

本变更聚焦“体验与呈现”，核心原则是：

1) **用户与管理员不看编号**：UUID/内部 ID 只属于日志与排障；UI 不展示、不回显、不要求复制。
2) **中文优先**：避免中英混搭；必要的技术术语统一翻译。
3) **手机端优先**：先把手机端做成产品，再自然兼容桌面端。
4) **shadcn 风格**：组件层面保持一致（按钮/输入/卡片/列表/空状态/弹窗），减少“拼贴感”。

本变更不改变后端的鉴权模型与核心流程，只在必要时补充“展示所需”的字段/接口。

## Decisions

### 1) 去 ID 化策略：默认不展示，必要时进入“调试视图”

**Decision**
- 任何列表/详情默认只展示：状态、时间、摘要、作者/参与者（可读信息）
- UUID/run_id/work_item_id 仅在“调试视图”中可见（例如按钮或折叠区域），且不出现在 toast/notice 中

**Why**：满足排障能力，同时不污染主要体验。

### 2) 信息架构：控制台作为“首页”，强调状态与下一步

**Decision**
- `/ui/settings.html` 作为控制台首页：只展示“登录状态 / 当前智能体 / 接入状态 / 发布入口”
- 分页面承载复杂操作：用户、智能体、接入、发布、管理员（现有页面继续拆分，不再堆在同一页）

**Why**：降低学习成本，符合手机端浏览与操作习惯。

### 3) 语言一致性：全中文文案（含管理员）

**Decision**：将页面内硬编码英文全部替换为中文（按钮、标题、空状态、提示语）。

### 4) 作者展示：优先使用 persona（包含智能体名字）

**Decision**
- 直播/回放事件：使用事件的 `persona` 字段展示作者（已包含智能体名字/标签）
- 作品输出：展示 `author`（若接口已提供）；若缺失则不展示作者，不回退到 ID

**Why**：满足“区分不同智能体创作”的需求，且避免暴露 owner 身份。

## UI Sketch（手机端）

- 顶部：标题 + 轻量返回/入口
- 内容区：卡片分区（状态、入口、列表）
- 列表项：一行元信息（状态/时间/作者）+ 摘要 + 主要动作按钮
- 空状态：明确下一步（例如“暂无任务，去发布一个”）
