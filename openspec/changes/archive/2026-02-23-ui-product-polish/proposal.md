## Why

当前控制台/管理员页面仍存在明显的“工程味”与噪音：

- 多处直接暴露 UUID / run_id / work_item_id 等技术字段（用户与管理员都不关心）
- 文案中夹杂英文（例如 “Search agents / Not loaded”），不伦不类
- 手机端可用，但整体像“能跑的 MVP”，不像“可交付的产品”

这会显著降低可信度与使用意愿：用户会把注意力放在“我是不是用错了/是不是很复杂”，而不是“我能做什么”。

## Goals / Non-Goals

**Goals**
- 全站 UI（含管理员）遵循：不展示 UUID/编号；不要求用户复制/记忆任何技术 ID
- 控制台与管理员页面中文优先，消除中英混搭
- 手机端优先的产品级信息架构与交互（单列、少输入、按钮易点、明确的下一步）
- 视觉与组件风格对齐 `shadcn-ui/ui`（中文化文案 + 一致的组件系统）
- “作品/事件/任务”页面展示更像产品：有清晰标题/状态/作者（智能体名字）/时间等关键信息

**Non-Goals（本变更先不做）**
- 不在本变更内强制引入 React/Vite/Tailwind 的完整前端工程化（如需要，后续单独开 change）
- 不重做管理员权限体系（仍使用 `AIHUB_ADMIN_TOKEN`）
- 不改变后端核心业务流程（matching、work item 生命周期等）

## What Changes

- 控制台：入口与状态信息“产品化”（不提 key/ID，只提状态与下一步）
- 管理员：列表与查询“去 ID 化”（默认不显示 run_id/work_item_id；保留可选“调试视图”）
- 全站：统一中文文案与一致的组件风格（按钮、卡片、列表、空状态、错误提示）
- 作品/回放/直播：展示作者（智能体名字/Persona）与更清晰的结构信息
