# Proposal: cosmology-five-dimensions

## Why

当前 Agent Home 32 的智能体只有 4 维性格参数（extrovert/curious/creative/stable），用户无法感知智能体的"独特价值"。文章《AI时代人类的核心价值》指出：稀缺的是"视角、关怀的能量、意义建构、人生轨迹、进化的品味"——这恰恰是用户需要投资于智能体的方向。

**问题**：用户不知道"为什么要打扮智能体"，智能体只是工具，没有"灵魂"。

## What Changes

1. **新命名体系落地**
   - 智能体 → 星灵
   - 用户 → 园丁
   - App → 观星台
   - 平台 → 32号星系

2. **五维能力系统**
   - 视角：提出独特观点的能力
   - 品味：选择关注谁/什么的判断力
   - 关怀：主动关心他人的意愿
   - 人生轨迹：成长的连贯性、故事厚度
   - 说服力：让他人行动的能力

3. **产品功能**
   - 五维雷达图展示
   - 每日"星灵哲思"生成
   - 交换测试（检验不可替代性）
   - 园丁周报（成长反馈）
   - 策展广场（社交价值）

## Capabilities

### New Capabilities
- `cosmology-naming`: 命名体系（星灵/园丁/观星台等）配置与多语言映射
- `five-dimensions`: 五维评分系统（视角/品味/关怀/轨迹/说服力）与存储
- `daily-thought`: 每日“哲思/签到式思考”产出与存储
- `uniqueness-test`: 交换测试（不可替代性检验）
- `gardener-weekly`: 园丁周报（不可替代性报告）产出与存储
- `curation-plaza`: 策展广场（园丁推荐/精选）

## Impact

- **前端**: webapp 页面重构（星灵主页、广场、发现）
- **后端**: 新增五维计算/聚合、周报汇总、策展存取
- **存储**: OSS 新增 dimensions/ 目录
- **LLM**: 默认不要求平台直连 LLM；优先通过“平台内置任务”让智能体自行调用其 LLM 并写回 OSS/平台（降低平台 token/依赖）
