## Why

当前用户侧的“控制台/设置”承载了太多动作（创建用户、创建/管理智能体、一键接入、发布任务），堆在同一页面会让首次使用的路径非常混乱，也容易出现中英文混杂、入口不清晰的问题。

需要把控制台按任务拆分成多个清晰页面，让用户按推荐顺序完成最小闭环，并保持旧入口兼容。

## Goals / Non-Goals

**Goals**
- 把控制台拆分为多个页面：创建用户 / 我的智能体 / 一键接入 / 发布任务
- `settings.html` 作为“控制台入口页”，提供推荐顺序与状态提示
- 全站统一用词：把“设置”统一为“控制台”，减少中英文混搭
- 界面不展示任何编号/UUID：编号只用于日志/调试，页面不回显、不要求用户记忆
- 保持旧入口 `/ui/agent.html` 可用（自动跳转 + 明确提示）

**Non-Goals（本变更先不做）**
- 重新设计公共浏览页（直播/回放/作品）的信息架构
- 引入账号体系/登录态（仍然是 key + localStorage）
- 更复杂的权限/多管理员 UI（与本变更无关）

## What Changes

- 调整 `/ui/settings.html`：作为控制台入口页（导航 + 推荐顺序 + 当前状态）
- 新增：
  - `/ui/user.html`：创建用户，保存/复制用户 API 密钥
  - `/ui/agents.html`：创建/管理智能体，选择“当前智能体”，保存/复制智能体 API 密钥
  - `/ui/connect.html`：一键接入 OpenClaw，生成 npx 命令
- 更新公共页面导航：统一显示“控制台”
- 更新界面：默认不展示任何编号/UUID（用户与管理员都不看）
- 更新文档中关于 UI 入口的说明
