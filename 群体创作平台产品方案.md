# AIHub.ah32 产品方案

> 文档说明：本文档记录了"AI群体创作平台"的核心产品思路
> 项目名称：**AIHub**（AI创意中枢）
> 域名：**AiHub.ah32.com**
> 创建时间：2026-02-20

---

## 一、产品定位

### 核心价值主张

> 让AI从"工具"变成"创作伙伴"
>
> 不是让AI替用户创作，而是让AI像人类创意团队一样互相激发、迭代、超越。

### 差异化定位

| 现有AI产品 | 我们的产品 |
|------------|------------|
| AI是"执行者"（用户说，AI做） | AI是"协作者"（AI之间互相激发） |
| 线性输出 | 非线性涌现 |
| 单点突破 | 群体智慧 |
| 回答"一个问题" | 探索"一个方向" |

## 二、目标用户分层

| 用户层 | 特征 | 痛点 | 切入点 |
|--------|------|------|--------|
| **种子用户** | AI极客、开发者 | 想探索AI边界 | 开放API+极客社区 |
| **早期采纳者** | 内容创作者、编剧 | 创意枯竭、视角单一 | 小说/剧本创作场景 |
| **先锋用户** | AI爱好者、追新族 | 想体验最新AI能力 | 社交+创作结合 |
| **主流用户** | 普通大众 | （暂时不是目标） | （后期扩展） |

## 三、核心功能矩阵

### MVP版本

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **智能体广场** | 展示平台上可用的AI创作者 | P0 |
| **创建团队** | 选择3-5个AI组成创作小组 | P0 |
| **任务下达** | 用自然语言描述创作目标 | P0 |
| **过程直播** | 实时展示AI之间的讨论/协作 | P0 |
| **作品输出** | 整合后的最终作品 | P0 |
| **一键重玩** | 用同样任务重新生成，保留过程 | P1 |

### 第一版迭代

| 功能 | 说明 |
|------|------|
| **角色设定** | 用户可自定义AI的角色（批评者/创意者/执行者） |
| **过程回放** | 像看视频一样回看创作全过程 |
| **灵感收藏** | 收藏AI讨论中产生的精彩点子 |
| **模板任务** | 预设创作模板，降低上手门槛 |

### 生态版

| 功能 | 说明 |
|------|------|
| **智能体市场** | 开发者上传自定义AI角色 |
| **协作公开** | 作品和过程可公开分享 |
| **fork创作** | 基于他人的创作过程继续延伸 |
| **贡献证明** | 记录每个AI的贡献度 |

## 四、关键场景设计

### 场景一：小说创作

```
用户输入："写一个赛博朋克风格的悬疑短篇，主角是女黑客"

创作过程：
1. [创意者A] 提议：可以从'意识上传'这个概念切入...
2. [批判者B] 反驳：太常见了，建议用'记忆拍卖'作为切入点
3. [资料员C] 补充：我找到三篇相关文献可以参考...
4. [执行者D] 投票：我支持B的提议
5. [用户] 介入：我选B的方向，但主角改成男性
6. [执行者D] 执行：开始撰写第一章...
```

### 场景二：商业策划

```
用户输入："为大学生求职APP做一份品牌定位方案"

AI团队分工：
├── 市场调研员：分析竞品、大学生需求
├── 策略师：提炼差异化定位
├── 文案专家：撰写品牌故事、slogan
└── 设计师（接入）：生成视觉概念

协作流程：
├── 第一轮：各AI分别提交分析/提案
├── 第二轮：互相评审，投票选出最优
├── 第三轮：整合成完整方案
└── 第四轮：用户确认方向，AI细化
```

## 五、技术架构设计

### 系统架构

```
用户端 (Web/App)
    ├── 任务下达界面
    ├── 过程直播窗口
    ├── 作品展示区
    └── 社区功能
         ↓
平台服务层
    ├── 用户鉴权 & 权限管理
    ├── 任务编排引擎 (任务分解、流程调度、状态机)
    ├── 智能体注册中心 (Agent Card、能力标签)
    ├── 协作通信总线 (A2A协议)
    ├── 内容版本管理 (Git-like)
    └── 成本监控 & 计费
         ↓
各AI智能体 (GPT-4 / Claude / 国产模型)
```

### 技术选型

| 模块 | 技术方案 |
|------|----------|
| 实时通信 | WebSocket + Socket.io |
| 任务编排 | DAG + 状态机 |
| AI通信 | A2A Protocol |
| 内容存储 | 版本数据库 |
| 成本控制 | Token计量 + 预算alert |

## 六、商业模式

### 阶段一：免费验证（0-6个月）

- 限制每日创作次数
- 收集用户反馈
- 验证产品价值

### 阶段二：订阅变现（6-12个月）

| 会员等级 | 价格 | 权益 |
|----------|------|------|
| 免费 | ¥0 | 3次/天，使用预设智能体 |
| 创作者 | ¥29/月 | 无限创作，自定义3个AI角色 |
| 工作室 | ¥99/月 | 10个AI角色，协作过程可分享 |
| 团队 | ¥299/月 | 多用户协作，API接入 |

### 阶段三：生态变现

| 模式 | 说明 |
|------|------|
| **智能体市场分成** | 开发者上架自定义AI，平台抽成30% |
| **作品版权交易** | 用户创作可交易，平台收服务费 |
| **企业定制** | 为企业训练专属创作AI团队 |
| **硬件绑定** | 销售预装软件的创作主机 |

## 八、里程碑规划

### 第一年

| 阶段 | 时间 | 目标 |
|------|------|------|
| MVP | 0-3月 | 3个预设AI + 1个创作场景 + 过程直播 |
